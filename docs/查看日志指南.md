# 查看服务器日志指南

## 日志分类

### 1. 应用运行日志（Docker 容器日志）
- **位置**：Docker 容器的标准输出/错误输出
- **内容**：Next.js 应用运行时的所有 `console.log`、`console.error`、`console.warn` 输出
- **包括**：
  - API 请求日志
  - 错误信息
  - 调试信息
  - 访问记录相关的日志（如 duration 更新、visitId 获取等）

### 2. 访问数据（不是日志，是数据文件）
- **位置**：`/opt/iridescent-blog/content/analytics.json`
- **内容**：访问记录数据（JSON 格式）
- **说明**：这是存储的访问数据，不是运行日志

## 查看 Docker 容器日志

### 基本命令

```bash
# 查看容器日志（实时跟踪，类似 tail -f）
docker logs -f iridescent-blog

# 查看最近 100 行日志
docker logs --tail 100 iridescent-blog

# 查看最近 10 分钟的日志
docker logs --since 10m iridescent-blog

# 查看指定时间段的日志
docker logs --since "2026-01-26T20:00:00" --until "2026-01-26T21:00:00" iridescent-blog

# 查看所有历史日志（从容器启动开始）
docker logs iridescent-blog

# 查看日志并包含时间戳
docker logs -t iridescent-blog
```

### 过滤特定日志

```bash
# 只查看错误日志
docker logs iridescent-blog 2>&1 | grep -i error

# 只查看包含 "duration" 的日志
docker logs iridescent-blog 2>&1 | grep -i duration

# 只查看包含 "visitId" 的日志
docker logs iridescent-blog 2>&1 | grep -i visitid

# 查看 API 相关日志
docker logs iridescent-blog 2>&1 | grep -i "api/analytics"

# 查看最近 50 行错误日志
docker logs --tail 200 iridescent-blog 2>&1 | grep -i error | tail -50
```

### 保存日志到文件

```bash
# 保存所有日志到文件
docker logs iridescent-blog > /tmp/app-logs-$(date +%Y%m%d-%H%M%S).log

# 保存最近 1000 行日志
docker logs --tail 1000 iridescent-blog > /tmp/recent-logs.log

# 保存错误日志
docker logs iridescent-blog 2>&1 | grep -i error > /tmp/error-logs.log
```

## 查看访问数据文件

访问数据存储在 JSON 文件中，不是日志格式：

```bash
# 查看访问数据文件
cat /opt/iridescent-blog/content/analytics.json

# 格式化查看（需要 jq 工具）
cat /opt/iridescent-blog/content/analytics.json | jq .

# 查看文件大小
ls -lh /opt/iridescent-blog/content/analytics.json

# 查看最近添加的访问记录（最后 10 条）
cat /opt/iridescent-blog/content/analytics.json | jq '.visits[-10:]'

# 查看特定路径的访问记录
cat /opt/iridescent-blog/content/analytics.json | jq '.visits[] | select(.path == "/about")'
```

## 常见日志内容说明

### 1. Duration 更新日志
```
Duration updated successfully: visitId=xxx, duration=300
```
- 表示停留时间更新成功
- `visitId`: 访问记录ID
- `duration`: 停留时间（秒）

### 2. VisitId 获取日志
```
Found visitId for path /post/xxx: visitId-xxx, time diff: 5000ms
```
- 表示成功找到访问记录ID
- `time diff`: 距离访问记录创建的时间差

### 3. 错误日志
```
Failed to parse request body: ...
Error updating duration: ...
Visit not found: ...
```
- 这些是错误信息，需要关注

### 4. 访问记录日志
```
Failed to record visit: ...
```
- 访问记录失败的错误信息

## 实时监控日志

### 方法 1：使用 docker logs -f
```bash
# 实时查看所有日志
docker logs -f iridescent-blog
```

### 方法 2：使用 tail 命令
```bash
# 如果日志被重定向到文件
tail -f /var/log/iridescent-blog.log
```

### 方法 3：使用 journalctl（如果使用 systemd）
```bash
# 如果容器通过 systemd 管理
journalctl -u iridescent-blog -f
```

## 调试停留时间问题

如果停留时间没有记录，可以按以下步骤排查：

```bash
# 1. 查看最近的 duration 相关日志
docker logs --tail 200 iridescent-blog 2>&1 | grep -i duration

# 2. 查看 visitId 获取日志
docker logs --tail 200 iridescent-blog 2>&1 | grep -i visitid

# 3. 查看错误日志
docker logs --tail 200 iridescent-blog 2>&1 | grep -i error

# 4. 查看 API 请求日志
docker logs --tail 200 iridescent-blog 2>&1 | grep "api/analytics"

# 5. 检查访问数据文件中的记录
cat /opt/iridescent-blog/content/analytics.json | jq '.visits[] | select(.path == "/post/xxx") | {id, duration, timestamp}'
```

## 日志文件位置总结

| 类型 | 位置 | 说明 |
|------|------|------|
| 应用运行日志 | Docker 容器标准输出 | 通过 `docker logs` 查看 |
| 访问数据 | `/opt/iridescent-blog/content/analytics.json` | JSON 格式的数据文件 |
| 构建日志 | 构建时输出 | 构建过程中的日志 |

## 注意事项

1. **日志不会自动保存到文件**：Docker 容器的日志默认只保存在 Docker 的日志驱动中，不会自动写入文件系统
2. **日志会占用磁盘空间**：如果日志过多，可能需要定期清理
3. **访问数据是数据文件**：`analytics.json` 是数据存储文件，不是日志文件
4. **生产环境日志**：生产环境的日志可能包含敏感信息，注意保护

## 清理日志

```bash
# 清理 Docker 容器日志（需要谨慎操作）
# 注意：这会删除所有历史日志
truncate -s 0 $(docker inspect --format='{{.LogPath}}' iridescent-blog)

# 或者限制日志大小（在 docker run 时添加）
docker run ... --log-opt max-size=10m --log-opt max-file=3 ...
```
