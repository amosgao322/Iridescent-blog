## 推荐技术方案

### 🎯 Next.js 14 + 后台管理系统

**为什么选择 Next.js：**
1. ✅ 支持全栈开发（前端 + API 路由）
2. ✅ 可以轻松实现文件上传和管理功能
3. ✅ 支持 Docker 部署
4. ✅ 性能优秀，SEO 友好
5. ✅ 生态丰富，可视化编辑器选择多

## 完整技术栈

### 前端部分
- **框架**：Next.js 14+ (App Router)
- **语言**：TypeScript
- **样式**：Tailwind CSS
- **Markdown 渲染**：MDX + remark/rehype
- **代码高亮**：shiki
- **搜索**：flexsearch（客户端搜索）

### 后台管理部分
- **编辑器**：MDX Editor 或 Toast UI Editor（Markdown 可视化编辑）
- **文件上传**：Next.js API Routes + multer/formidable
- **认证**：NextAuth.js（可选，如果只需要个人使用可以简化）

### 部署部分
- **容器化**：Docker + Docker Compose
- **Web 服务器**：Nginx（反向代理）
- **Node.js 运行时**：Node.js 20 LTS

### 2. 文件存储方式
- [ ] **选项 A**：本地存储（推荐，简单）
  - Markdown 文件存储在服务器 `content/` 目录
  - 图片存储在服务器 `content/images/` 或 `public/images/`
  - 使用 Docker volume 持久化
  
- [ ] **选项 B**：对象存储（OSS/S3）
  - 图片上传到阿里云 OSS 或 AWS S3

  - 需要配置 OSS 密钥

**推荐选项 A**，个人博客够用，后续可升级到 OSS。

- [ ] **选项 C**：无需认证（不推荐）
