# Iridescent Blog 部署指南

## 前置准备

### 1. 服务器要求
- 操作系统：Linux (推荐 Ubuntu 20.04+ 或 CentOS 7+)
- 内存：至少 2GB
- 磁盘空间：至少 10GB
- 已安装 Docker 和 Docker Compose

### 2. 安装 Docker 和 Docker Compose

#### Ubuntu/Debian
```bash
# 更新包索引
sudo apt-get update

# 安装 Docker
sudo apt-get install -y docker.io docker-compose

# 启动 Docker 服务
sudo systemctl start docker
sudo systemctl enable docker

# 验证安装
docker --version
docker-compose --version
```

#### CentOS/RHEL
```bash
# 安装 Docker
sudo yum install -y docker docker-compose

# 启动 Docker 服务
sudo systemctl start docker
sudo systemctl enable docker

# 验证安装
docker --version
docker-compose --version
```

## 部署方式：本地构建镜像后上传

这种方式适合第一次部署，可以让你更好地理解整个流程。

**为什么选择这种方式？**
- 本地构建速度快，不受服务器性能限制
- 可以提前测试镜像是否正常
- 减少服务器资源占用
- 构建过程在本地完成，便于排查问题

---

## 第一步：在本地构建 Docker 镜像

### 1.1 进入项目目录

```bash
# 在项目根目录执行（Windows PowerShell 或 Linux/Mac 终端）
cd D:\dream-start\code\Iridescent-blog  # Windows 示例路径
# 或
cd /path/to/Iridescent-blog  # Linux/Mac 示例路径
```

**说明：** 确保你在项目根目录，这样 Docker 才能正确找到所有文件。

### 1.2 构建 Docker 镜像

```bash
# 进入 docker 目录
cd docker

# 构建镜像
docker build -f Dockerfile -t iridescent-blog:latest ..
```

**命令解释：**
- `docker build` - Docker 构建命令
- `-f Dockerfile` - 指定 Dockerfile 文件位置（在 docker 目录下）
- `-t iridescent-blog:latest` - 给镜像打标签，`iridescent-blog` 是镜像名，`latest` 是版本标签
- `..` - 构建上下文路径，`..` 表示上一级目录（项目根目录），因为 Dockerfile 在 docker 子目录中

**说明：** Dockerfile 默认使用国内镜像源（如 USTC）拉取 `node:20-alpine`，避免直连 Docker Hub 超时。若需直连 Docker Hub，可加参数：`--build-arg BASE_IMAGE=node:20-alpine`。

**构建过程说明：**
1. Docker 会读取 Dockerfile，按照指令逐步构建
2. 首先安装依赖（`npm ci`）
3. 然后构建 Next.js 应用（`npm run build`）
4. 最后创建运行时的精简镜像
5. 整个过程可能需要 5-15 分钟，取决于你的网络和机器性能

### 1.3 验证镜像构建成功

```bash
# 查看本地所有镜像，筛选出我们刚构建的
docker images | grep iridescent-blog
```

**预期输出：**
```
iridescent-blog   latest   abc123def456   2 minutes ago   800MB
```

**说明：** 
- 如果能看到镜像，说明构建成功
- 镜像大小通常在 500MB-2GB 之间，这是正常的（包含了 Node.js 运行时和你的应用）

## 第二步：导出镜像为文件

### 2.1 导出镜像

```bash
# 在项目根目录或任意位置执行
docker save iridescent-blog:latest -o iridescent-blog.tar
```

**命令解释：**
- `docker save` - 将镜像保存为 tar 文件
- `iridescent-blog:latest` - 要导出的镜像名称和标签
- `-o iridescent-blog.tar` - 输出文件名，`-o` 表示 output

**说明：**
- 这个命令会将整个镜像（包括所有层）打包成一个 tar 文件
- 文件会保存在当前目录
- 文件可能很大（500MB-2GB），这是正常的

### 2.2 查看文件大小

**Windows PowerShell:**
```powershell
Get-Item iridescent-blog.tar | Select-Object Name, @{Name="Size(MB)";Expression={[math]::Round($_.Length/1MB,2)}}
```

**Linux/Mac:**
```bash
ls -lh iridescent-blog.tar
```

**说明：** 确认文件已生成，并记录文件大小，这样上传时心里有数。

### 2.3 （可选）压缩镜像文件

如果文件很大，可以压缩后再上传，节省传输时间：

**Linux/Mac:**
```bash
gzip iridescent-blog.tar
# 会生成 iridescent-blog.tar.gz，通常能压缩到原来的 30-50%
```

**Windows:**
- 可以使用 7-Zip 或其他压缩工具压缩 `iridescent-blog.tar`

**说明：** 压缩是可选的，如果网络很快可以不压缩。

## 第三步：上传镜像文件到服务器

### 3.1 使用 SCP 上传

**如果未压缩：**
```bash
scp iridescent-blog.tar user@your-server-ip:/opt/
```

**如果压缩了：**
```bash
scp iridescent-blog.tar.gz user@your-server-ip:/opt/
```

**命令解释：**
- `scp` - Secure Copy，通过 SSH 安全传输文件
- `iridescent-blog.tar` - 本地文件路径
- `user@your-server-ip:/opt/` - 目标位置，`user` 是你的服务器用户名，`your-server-ip` 是服务器 IP 地址

**说明：**
- 上传时间取决于文件大小和网络速度
- 如果网络中断，可以使用 `rsync` 命令支持断点续传（见下方）

### 3.2 （可选）使用 rsync 上传（支持断点续传）

```bash
rsync -avz --progress iridescent-blog.tar user@your-server-ip:/opt/
```

**命令解释：**
- `rsync` - 更强大的文件同步工具
- `-avz` - 参数：`a`(归档模式), `v`(详细输出), `z`(压缩传输)
- `--progress` - 显示传输进度

**说明：** 如果文件很大或网络不稳定，推荐使用 rsync。

## 第六步：准备项目配置文件

### 6.1 创建必要的目录结构

```bash
# 在 /opt/iridescent-blog 目录下
mkdir -p docker/nginx/ssl

```

**说明：**

- `content/posts` - 存放博客文章
- `content/notes` - 存放笔记

### 6.2 创建环境变量文件

```bash
nano .env
```

**说明：** `nano` 是简单的文本编辑器，你也可以使用 `vi` 或 `vim`。

在编辑器中添加以下内容：

```env
# 管理员密码（用于访问 /admin 页面）
ADMIN_PASSWORD=your_secure_password_here

# 腾讯云 COS 配置（用于图片上传）
COS_SECRET_ID=your_cos_secret_id
COS_SECRET_KEY=your_cos_secret_key
COS_BUCKET=your_bucket_name
```

**重要说明：**
- 将 `your_secure_password_here` 替换为强密码
- 填入你的腾讯云 COS 凭证信息
- 保存文件：在 nano 中按 `Ctrl+O` 保存，`Ctrl+X` 退出

### 6.3 设置文件权限

```bash
chmod 600 .env
```

**说明：** 
- `600` 权限表示只有文件所有者可以读写，其他人无法访问
- 这是安全最佳实践，防止敏感信息泄露

## 第七步：创建 docker-compose.yml 文件

### 7.1 创建 docker-compose.yml

```bash
nano docker-compose.yml
```

### 7.2 添加配置内容

```yaml
version: '3.8'

services:
  # 博客应用服务
  blog:
    image: iridescent-blog:latest  # 使用我们导入的镜像
    container_name: iridescent-blog
    ports:
      - "3000:3000"  # 将容器的 3000 端口映射到主机的 3000 端口
    environment:
      - NODE_ENV=production
      # 从 .env 文件读取环境变量
      - ADMIN_PASSWORD=${ADMIN_PASSWORD:-admin123}
      - COS_SECRET_ID=${COS_SECRET_ID}
      - COS_SECRET_KEY=${COS_SECRET_KEY}
      - COS_BUCKET=${COS_BUCKET}
      - COS_REGION=${COS_REGION:-ap-guangzhou}
    volumes:
      # 将本地的 content 目录挂载到容器中，这样文章数据会持久化
      - ./content:/app/content
    restart: unless-stopped  # 容器异常退出时自动重启
    networks:
      - blog-network  # 连接到自定义网络

  # Nginx 反向代理服务
  nginx:
    image: nginx:alpine  # 使用轻量级的 Nginx 镜像
    container_name: iridescent-blog-nginx
    ports:
      - "86:86"    # HTTP 端口
      - "443:443"  # HTTPS 端口（如果配置了 SSL）
    volumes:
      # 挂载 Nginx 配置文件
      - ./docker/nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      # 挂载 SSL 证书目录
      - ./docker/nginx/ssl:/etc/nginx/ssl:ro
    depends_on:
      - blog  # 确保 blog 服务先启动
    restart: unless-stopped
    networks:
      - blog-network

# 定义网络
networks:
  blog-network:
    driver: bridge  # 使用桥接网络，容器之间可以互相通信
```

**配置说明：**
- `services` - 定义要运行的服务（容器）
- `blog` - 你的 Next.js 应用
- `nginx` - 反向代理，将外部请求转发到 blog 服务
- `networks` - 定义网络，让容器可以互相通信
- `volumes` - 数据持久化，确保文章数据不会丢失

## 第八步：创建 Nginx 配置文件

### 8.1 创建配置文件

```bash
nano docker/nginx/nginx.conf
```

### 8.2 添加配置内容

```nginx
events {
    worker_connections 1024;
}

http {
    # 定义后端服务（你的 Next.js 应用）
    upstream nextjs {
        server blog:3000;  # blog 是 docker-compose 中的服务名
    }

    # HTTP 服务器配置
    server {
        listen 86;
        server_name _;  # _ 表示接受所有域名

        # 将所有请求转发到 Next.js 应用
        location / {
            proxy_pass http://nextjs;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection 'upgrade';
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_cache_bypass $http_upgrade;
        }

        # 静态文件缓存（提高性能）
        location /_next/static {
            proxy_pass http://nextjs;
            proxy_cache_valid 200 60m;
            add_header Cache-Control "public, immutable";
        }
    }

    # HTTPS 配置（可选，如果需要 HTTPS 可以取消注释）
    # server {
    #     listen 443 ssl http2;
    #     server_name your-domain.com;
    #
    #     ssl_certificate /etc/nginx/ssl/cert.pem;
    #     ssl_certificate_key /etc/nginx/ssl/key.pem;
    #
    #     location / {
    #         proxy_pass http://nextjs;
    #         proxy_http_version 1.1;
    #         proxy_set_header Upgrade $http_upgrade;
    #         proxy_set_header Connection 'upgrade';
    #         proxy_set_header Host $host;
    #         proxy_set_header X-Real-IP $remote_addr;
    #         proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    #         proxy_set_header X-Forwarded-Proto $scheme;
    #         proxy_cache_bypass $http_upgrade;
    #     }
    # }
}
```

**配置说明：**
- `upstream nextjs` - 定义后端服务地址
- `proxy_pass` - 将请求转发到后端
- `proxy_set_header` - 设置请求头，让后端知道真实的客户端信息
- 静态文件缓存可以提高网站加载速度

## 第九步：启动服务

### 9.1 启动容器

```bash
# 确保在 /opt/iridescent-blog 目录
cd /opt/iridescent-blog

# 启动所有服务（-d 表示后台运行）
docker-compose up -d
```

**命令解释：**
- `docker-compose up` - 根据 docker-compose.yml 启动所有服务
- `-d` - detached 模式，后台运行

**说明：**
- Docker Compose 会读取 docker-compose.yml，创建网络，启动两个容器
- 第一次启动可能需要几秒钟

### 9.2 查看容器状态

```bash
docker-compose ps
```

**预期输出：**
```
NAME                      STATUS              PORTS
iridescent-blog           Up 30 seconds       0.0.0.0:3000->3000/tcp
iridescent-blog-nginx     Up 30 seconds       0.0.0.0:80->80/tcp, 0.0.0.0:443->443/tcp
```

**说明：**
- `STATUS` 显示 `Up` 表示容器正在运行
- 如果显示其他状态，需要查看日志排查问题

### 9.3 查看日志

```bash
# 查看所有服务的日志
docker-compose logs -f
```

**说明：**
- `-f` 表示 follow，实时显示日志
- 按 `Ctrl+C` 退出日志查看
- 如果看到错误信息，可以根据错误提示排查

## 更新部署

当你修改了代码，需要重新部署时：

### 1. 在本地重新构建镜像

```bash
# 在本地项目目录
cd docker
docker build -f Dockerfile -t iridescent-blog:latest ..
```

### 2. 导出新镜像

```bash
docker save iridescent-blog:latest -o iridescent-blog.tar
```

### 3. 上传到服务器

```bash
scp iridescent-blog.tar user@your-server-ip:/opt/
```

### 4. 在服务器上更新

```bash
# SSH 到服务器
ssh user@your-server-ip

# 停止旧容器
cd /opt/iridescent-blog
docker-compose down

# 导入新镜像
docker load -i /opt/iridescent-blog.tar

# 启动新容器
docker-compose up -d

# 清理旧镜像（可选，释放磁盘空间）
docker image prune -f
```

## 重要注意事项

1. **环境变量安全**：`.env` 文件包含敏感信息，不要提交到 Git


### 日志说明

- **应用运行日志**：通过 `docker logs` 查看，包含所有 `console.log`、`console.error` 输出
- **访问数据**：存储在 `/opt/iridescent-blog/content/analytics.json`（这是数据文件，不是日志）
- **详细日志查看指南**：参考 `docs/查看日志指南.md`

### 常见日志内容

- `Duration updated successfully`: 停留时间更新成功
- `Found visitId for path`: 成功找到访问记录ID
- `Failed to parse request body`: 请求解析失败（错误）
- `Visit not found`: 访问记录未找到（错误）
