# 检查步骤

## 1. 查看 Nginx 配置文件内容

```bash
# 查看配置文件内容
sudo cat /etc/nginx/sites-available/iridescent-blog

# 或者只查看 HTTPS server 块
sudo grep -A 50 "listen 443" /etc/nginx/sites-available/iridescent-blog
```

## 2. 检查访问记录（不使用 jq）

```bash
# 查看最后几条访问记录
docker exec iridescent-blog sh -c "tail -100 /app/content/analytics.json | grep -o '\"ip\":\"[^\"]*\"' | tail -10"

# 查看最近的访问记录（包括路径和时间戳）
docker exec iridescent-blog sh -c "tail -200 /app/content/analytics.json | grep -E '\"ip\"|\"path\"|\"timestamp\"' | tail -30"
```

## 3. 实时监控日志

```bash
# 在一个终端窗口执行
docker logs iridescent-blog -f | grep -E "Middleware|RecordVisit|recordVisit"
```

然后在浏览器中访问网站，观察日志输出。

## 4. 检查 Nginx 配置的关键点

请确认配置文件中 HTTPS server 块的每个 location 都包含：
- `proxy_set_header X-Real-IP $remote_addr;`
- `proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;`
- `proxy_set_header X-Forwarded-Proto https;` （必须是 `https`，不能是 `$scheme`）
